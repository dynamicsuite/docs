View Structure

A package may contain "views" which are content that the user sees on their screen.

Views are JSON objects with the following structure:

Note, if a parameter is not provided, the default will be used.

Parameters

The entry point of the view. This is the path to a PHP script which will be executed when the view is requested. Alternately, you can direct load a single Vue component (for SFCs) with the following syntax: 
vue://*component-name*
 The HTML title of the view. If not provided or NULL, the view ID will be used. If the view is public and does not require authentication. If the view is navigable from the overlay nav bar. If the overlay should be hidden for the view. The name of the view as appears on the overlay nav bar. If not given or NULL, the view ID will be used. The FontAwesome icon class to use on the overlay nav bar for the view. The nav group that the view is assigned to. An array of strings of required permissions on the session for the view. An array of string paths to include in the autoloader for the view. An array of string paths of PHP scripts to execute for the view. These scripts are executed before the entry. An array of string paths to JS files to include for the view. An array of string paths to CSS files to include for the view.
